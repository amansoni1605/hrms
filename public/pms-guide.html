1. What is the PMS Engine?

The Performance Management System (PMS) is the evaluation and people-development engine of the HRMS. It governs the entire employee performance lifecycle — from setting goals at the start of a cycle to approving salary revisions at the end.

🎯

Appraisal Cycles

Annual, half-year, quarterly, probation

Define phases, timelines, and formula weights for each evaluation period.

⭐

Weighted Scoring

Self 20% · Manager 60% · Peer 20%

Configurable formula. Final score aggregated across reviewer roles on a 1–5 scale.

💰

Comp Approval

Two-step with skip-level + SoD

Promotions and large increments require skip-level endorsement before HR sign-off.

🚨

PIP Auto-Trigger

Below-threshold score → PIP draft

When HR approves a cycle, employees scoring below the PIP threshold get auto-enrolled.

💬

Encrypted Feedback

AES-256-GCM at rest

Shoutouts, coaching notes, and one-on-one actions — all encrypted before storage.

🗂️

OKR Goal Tracking

Progress check-ins, key results

Goals linked to appraisal cycles. Track % progress with key results per goal.

2. Who Uses It

🛡️

HR Admin / Super Admin

Full control — create cycles, calibrate scores, approve compensation, manage PIPs

  • Create and configure appraisal cycles (phases, formula weights, calibration bands)
  • Advance cycle through its 8-phase state machine
  • Bulk-open self-reviews for all active employees
  • Calibrate individual scores (override final score)
  • Run score computation engine across all reviews
  • Give final compensation sign-off (last step in two-step flow)
  • Create and manage PIPs; view encrypted PIP notes
  • View all feedback events (decrypted)
  • Update org hierarchy (effective-dated reporting lines)
👔

Manager / HR Manager

Review direct reports, write ratings, recommend compensation

  • Access reviews for all direct reports via My Team
  • Rate each competency and goal dimension (1–5 scale)
  • Write encrypted narrative comments per rating
  • Finalize review — triggers compensation recommendation prompt
  • Recommend increment % and promotion (subject to skip-level / HR approval)
  • Act as skip-level manager for second-line reports' two-step flows
  • Add coaching notes and one-on-one actions via Feedback Engine
👤

Employee

Complete self-appraisal, set goals, view final review, acknowledge outcome

  • Open self-review at /my/performance during self-appraisal phase
  • Rate own competencies, summarize achievements & challenges
  • Submit self-assessment (cannot edit after submission)
  • Create and track OKR goals; add check-in progress updates
  • View final review once HR finalizes
  • Acknowledge outcome (marks review as acknowledged)
  • Give and receive anytime feedback and shoutouts

3. Core Concepts

Appraisal Cycle

A cycle is the top-level container for a performance period (e.g., "H1 2026 Annual Review"). It defines which phases run, when they open/close, the scoring formula, calibration bands, and the PIP score threshold.

PMS Review vs. Performance Review

ℹ️

Two review models co-exist: The newer PMS Review (ws_pms_reviews) supports multi-role, multi-phase cycles with encrypted comments, calibration, and 360°. The legacy Performance Review (ws_performance_reviews) is a simpler 5-status flow used by the /performance HR page. Both are actively used — the PMS model for new cycles, the legacy model for the comp-approval engine.

Phase Locking Behavior

⚠️

When advancing past self_appraisal, all self-reviews still in draft are automatically locked (lockedAt = now). Locked reviews cannot be edited. A manager can still call "recall" to reopen a locked review if needed, but this requires HR approval. The same lock applies to manager reviews when advancing past manager_review.

Creating a Cycle (Step by Step)

  1. Navigate to PMS → Admin Panel

    Go to /performance/pms → click the Admin Panel tab.

  2. Create New Cycle

    Click New Cycle. Enter name (e.g., "H1 2026"), type (annual/half_year/quarterly/probation), start/end dates.

  3. Configure Phases

    Set opensAt and closesAt dates for each phase. Add gracePeriodHours if late submissions are allowed.

  4. Set Formula Weights

    Configure component weights: self (20%), manager (60%), peer_avg (20%). Weights must total 100. Set scale min/max (default 1–5).

  5. Configure Calibration Bands

    Example: Outstanding (top 10%), Exceeds (next 20%), Meets (next 40%), Below (next 20%), Poor (bottom 10%).

  6. Set PIP Threshold

    Any calibrated score below this value automatically creates a PIP draft on HR approval. Set 0 to disable auto-PIP.

  7. Initiate Cycle

    Click Initiate. The system sends notifications and bulk-creates self-reviews for all active/on-leave employees.

6. Compensation Approval Flow

After a manager finalizes a review, they can recommend a salary increment or promotion. The system routes this through one or two approval steps depending on size and type.

When Two-Step Applies

⚠️

Two-step approval is required when: promotion = true OR increment % > configured threshold (e.g., ≥ 20%). Single-step (HR only) applies for standard merit increments.

Two-Step Flow

1

Manager Recommends

Sets increment %, promotion flag, proposed title/band, justification, effective date. System identifies skip-level manager from org hierarchy and classifies recommender relationship.

→
2

Skip-Level Endorsement

The 2nd-line manager receives a notification. They can approve (advances to HR) or reject (ends the chain). Segregation of duty: skip-level ≠ recommender.

→
3

HR Final Sign-off

HR Admin receives notification. Approves or rejects. Must differ from skip-level endorser. On acceptance: salary revision applied (or staged for future effective date) and audit log written.

Segregation of Duties (SoD) Rules

RuleEnforcement
Recommender ≠ ApproverThe manager who recommended cannot approve at any step
Skip-level endorser ≠ HR approverThe second-line manager who endorsed step 1 cannot give final sign-off
Only designated skip-level or super_admin can endorse step 1System validates endorser's employee ID against the stored skipLevelManagerId
Only hr_admin or super_admin can approve final stepRole check enforced in API handler

Compensation Decision States

none → pending → accepted or rejected

Compensation History (Audit Log)

When a compensation decision is accepted, an immutable record is written to ws_compensation_history. Old and new salary are AES-256-GCM encrypted. The increment %, titles, bands, effective date, and approver are stored in clear for reporting. The record can never be deleted or modified. 

// Example compensation history entry
{
  reviewId:      "ObjectId",
  cycleLabel:    "H1 2026",
  changeType:    "promotion",   // merit | promotion | market | adjustment
  incrementPct:  22.5,
  promotion:     true,
  oldTitle:      "Senior Engineer",
  newTitle:      "Staff Engineer",
  effectiveDate: "2026-07-01",
  appliedAt:     "2026-07-01T00:00:00Z",
  decidedById:   "UserId of HR approver",
  oldSalaryEnc:  "<Buffer — AES-256-GCM>",
  newSalaryEnc:  "<Buffer — AES-256-GCM>"
}

8. HR Admin — Step-by-Step Guide

  1. Create a Cycle

    Go to /performance/pms → Admin Panel. Click New Cycle. Fill in name, type, dates, formula weights, calibration bands, PIP threshold. Save as draft.

  2. Initiate the Cycle

    Click Initiate Cycle. The system auto-creates a self-review for every active/on-leave employee and sends notifications.

  3. Monitor Self-Appraisal Progress

    Use the Admin Panel to see who has not submitted. Send bulk reminders. View lagging employees list sorted by days remaining.

  4. Advance to Manager Review

    When satisfied with self-review completion rate, click Close Self-Appraisal. All draft self-reviews are auto-locked. Managers receive notifications to complete their reviews.

  5. (Optional) Advance to 360° Peer Review

    If the cycle has enable360=true, advance to peer_360. Employees nominate peers; nominated peers receive review requests.

  6. Calibrate Scores

    In the Calibration Grid, view the 9-box distribution. Override individual calibrated scores where needed. Use the distribution bands to ensure the org curve is healthy.

  7. Run Score Computation

    Click Compute Scores. The engine aggregates all submitted reviews, applies formula weights, and stamps finalScore on each review.

  8. Approve Cycle (HR Sign-off)

    Advance to approved_hr. Employees below the PIP threshold automatically receive a PIP draft. Review and activate each PIP.

  9. Process Compensation Approvals

    Review pending comp recommendations in /performance. For two-step flows, skip-level endorsement will have already arrived. Give final HR sign-off. On accept, salary revision is applied immediately or staged for the effective date.

  10. Sign Off & Archive

    Advance to signed_off so employees can view and acknowledge. Once all acknowledgements are in, archive the cycle.

9. Manager — Step-by-Step Guide

  1. Receive Notification

    When the cycle advances to manager_review, you receive an in-app notification and email. Go to /my/team or /performance.

  2. Open a Direct Report's Review

    In /performance, filter by status manager_review. Click on an employee row to open their review. You can see their self-assessment, goals, and your rating area.

  3. Rate Competencies

    For each of the 5 competencies (Delivery, Collaboration, Expertise, Ownership, Communication), assign a rating 1–5 and write a comment. Comments are encrypted at rest.

  4. Finalize Review

    Click Finalize Review. This saves all ratings and advances the review to manager_review status. You will be prompted to recommend a compensation adjustment.

  5. Submit Compensation Recommendation (Optional)

    Toggle on Recommend Compensation. Enter increment %, check Promotion if applicable, set proposed title/band and justification. Choose effective date. Submit — the two-step flow will activate if threshold is crossed.

  6. Act as Skip-Level Endorser

    For your second-line reports (your direct report's direct reports), you may receive a skip-level endorsement request. Review the recommendation and approve or reject in /performance.

10. Employee — Step-by-Step Guide

  1. Receive Review Notification

    When a cycle opens, you get an in-app notification and email. Click the link or go to /my/performance.

  2. Complete Self-Assessment

    Write a career summary, list your key achievements and challenges. Rate yourself on all 5 competencies. Add comments for each rating.

  3. Submit Self-Review

    Click Submit. Once submitted you cannot edit. If the deadline passes before you submit, the review is auto-locked by the system.

  4. Set Goals (OKRs)

    Go to /my/performance → My Goals. Add goals across Financial, Customer, Operational, or Learning & Growth perspectives. Each goal needs key results with targets.

  5. Track Progress

    Add check-in updates to your goals during the cycle. Update progressPct and record what changed since the last check-in.

  6. View Final Review

    Once the cycle reaches signed_off, you can see your final score, manager ratings, and calibrated score. Check your compensation outcome if applicable.

  7. Acknowledge the Review

    Click Acknowledge. You may add a comment. This marks the review as acknowledged — the final state in the cycle.

Goals are linked to appraisal cycles and organized into four strategic perspectives. Total weight across all goals must equal 100%.

Each goal has multiple key results with a target value, current value, unit of measurement (%, INR, USD, Score, etc.) and a done flag. Progress % is computed from key result completion.

13. Feedback Engine

The feedback engine supports four types of people-development events. All body content is AES-256-GCM encrypted before being written to the database.

🎉

Shoutout

Public recognition for a peer

Visibility: public. Can be linked to a specific cycle or goal.

💬

Anytime Feedback

Real-time developmental input

Visibility: private or manager_visible. Sentiment: positive / constructive / neutral.

📋

One-on-One Action

Action items from 1:1 meetings

Manager-to-employee. Tracks agreed action items from recurring check-ins.

🎓

Coaching Note

Private coaching record

Visibility: private. Written by manager or HR. Encrypted manager/HR notes per PIP checkpoint.

14. Score Computation Engine

The score engine is triggered by HR via POST /api/ws/performance/cycles/[id]/compute-scores.

  1. Aggregate Per-Role Averages

    For each (revieweeId, reviewerRole) pair, average all rating scores across submitted/locked reviews in the cycle.

  2. Collapse Peer Reviews

    Multiple peer reviews for the same reviewee are averaged into a single peer_avg value.

  3. Apply Formula Weights

    finalScore = (self_avg × self_weight + manager_avg × manager_weight + peer_avg × peer_weight) / totalWeight

  4. Clamp to Scale

    Ensure score is within [scale.min, scale.max]. Default: 1–5.

  5. Stamp finalScore

    Write finalScore to all submitted/locked reviews in the cycle. HR can then manually set calibratedScore for individual outliers.

// Default formula config stored on cycle
{
  components: [
    { source: "self",     weight: 20 },
    { source: "manager",  weight: 60 },
    { source: "peer_avg", weight: 20 }
  ],
  scale: { min: 1, max: 5 }
}

17. Encryption at Rest

All narrative content (review comments, feedback, PIP notes, salary figures) is encrypted at rest using AES-256-GCM with per-tenant data encryption keys (DEKs).

FieldModelFormat
ratings[].commentEnc + commentIvWorkspacePMSReview[authTag(16) | ciphertext] + separate iv(12)
overallCommentEnc + overallCommentIvWorkspacePMSReview[authTag(16) | ciphertext] + separate iv(12)
bodyEnc + bodyIvWorkspaceFeedbackEvent[authTag(16) | ciphertext] + separate iv(12)
checkpoints[].managerNotesEncWorkspacePIP[iv(12) | authTag(16) | ciphertext]
oldSalaryEnc / newSalaryEncWorkspaceCompensationHistoryAES-256-GCM encrypted salary Buffer
🔒

Encryption is handled transparently by the server. When reading a review, the API calls getTenantDEK(tenantId) to retrieve the tenant's key, then decrypts all encrypted fields before returning the JSON response. Clients always receive plaintext — they never see the raw encrypted buffers.

18. Effective-Dating (Org Hierarchy)

The WorkspaceOrgNode model tracks the organizational reporting structure with full historical accuracy using effective-dating. 

// When a manager change occurs:

// 1. Close the current active node
await WorkspaceOrgNode.updateOne(
  { employeeId, effectiveTo: null },
  { effectiveTo: new Date() }          // seal with current timestamp
);

// 2. Create the new node
await WorkspaceOrgNode.create({
  employeeId,
  ancestorPath:  [newManager, skip, ceo],  // chain up to root
  effectiveFrom: new Date(),
  effectiveTo:   null,                    // null = currently active
  depth:         3,
  departmentId,
  matrixManagerIds: []
});

This allows the compensation engine to call getManagementChain(employeeId) and reconstruct the exact reporting chain at any historical date — critical for auditing comp approval routing decisions after the fact.
